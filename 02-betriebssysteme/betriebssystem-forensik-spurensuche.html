Kurzueberblick

Betriebssysteme protokollieren mehr, als vielen Nutzern bewusst ist. Als Forensiker musst du die wichtigsten Artefakt-Fundorte in Windows, Linux und macOS kennen — und wissen, welche Tools sie auslesen.

Windows-Artefakte im Detail

Prefetch — Programmausfuehrung protokolliert

MerkmalBeschreibung
FundortC:\Windows\Prefetch\*.pf
FormatBinär (Version 17 unter WinXP, Version 23/26/30 unter Win7-11)
Enthaltene DatenExe-Name + Hash (8 Zeichen), Run Count (bis v26: 8x), letzte 8 Ausfuehrungszeitpunkte, Liste referenzierter Dateien (bis 100 Eintraege), geladene DLLs
NamensschemaCMD.EXE-89305D01.pf — Hash des Pfades (nicht der Datei!)
ToolsPECmd.exe (Zimmerman), WinPrefetchView (Nirsoft)
Forensische Perle: Der Prefetch enthaelt die vollstaendigen Pfade referenzierter DLLs und Dateien — selbst wenn die Originaldateien geloescht wurden. Beispiel: MALWARE.EXE-XXXXXXXX.pf listet unter DEVICE\HARDDISKVOLUME3\USERS\VICTIM\DESKTOP\MALWARE.EXE — inklusive Vornutzername!

Jump Lists — pro Anwendung zuletzt geoeffnete Dateien

TypFundortFormat
AutomaticDestinations%AppData%\Microsoft\Windows\Recent\AutomaticDestinations\*.automaticDestinations-msOLE Compound File (strukturiertes Binary)
CustomDestinations%AppData%\Microsoft\Windows\Recent\CustomDestinations\*.customDestinations-msSparsamer; von der App selbst gepflegt
  • App-ID: Die Dateinamen sind AppUserModelIDs (z.B. 1b4dd67f29cb1962 = Notepad)
  • DestList-Stream: Sortierte Liste von MRU-Objekten (Most Recently Used) — Reihenfolge + MAC-Zeiten
  • LNK-Streams: Enthalten Serialisierte Shell-Links mit Zielpfad, Argumenten, Icon-Location
  • Tools: JLECmd.exe (Zimmerman), JumpLister

LNK-Dateien — Shortcut als Beweismittel

# LNK = Windows Shell Link (Binary Format)
# Fundort: %AppData%\Microsoft\Windows\Recent\*.lnk
#           %AppData%\Microsoft\Office\Recent\*.lnk

# Forensich relevante Felder:
# ┌─────────────────────────────────────────────────────────────┐
# │ NetBIOS-Name des Servers    │ Bei Netzwerkfreigaben!       │
# │ MAC-Adresse des Hosts       │ Bei lokalen Dateien          │
# │ Volume Serial Number        │ Identifiziert das Volume     │
# │ Volume Label                │ Name des Laufwerks           │
# │ MAC-Zeiten der Zieldatei    │ Creation, Access, Modified   │
# │ Ziel-Pfad (langer+kurzer)   │ Absoluter Pfad zur Datei     │
# │ Icon-Location + Hotkey      │ Welches Icon, welcher Shortcut│
# │ Distributed Link Tracker    │ Objekt-ID + Birth Volume ID  │
# └─────────────────────────────────────────────────────────────┘
DLT (Distributed Link Tracker): Jede LNK-Datei enthaelt eine Object-ID und Birth-Volume-ID des Ziels. Selbst wenn die Zieldatei verschoben oder umbenannt wurde, kann ihre Herkunft ueber die Volume-ID zurueckverfolgt werden. Diese IDs finden sich auch im $STANDARD_INFORMATION-Attribut der MFT.

Tools: LECmd.exe (Zimmerman), lnk_parser.py

Shellbags — Fensterpositionen als Nutzungsbeweis

Shellbags speichern die Fenstereinstellungen (Groesse, Position, Sortierung) fuer jeden vom Explorer besuchten Ordner — auch nach Loeschung des Ordners.

Registry-KeyInhalt
NTUSER.DAT\Software\Microsoft\Windows\Shell\BagMRUHierarchische Ordnerstruktur (Nested MRU)
NTUSER.DAT\Software\Microsoft\Windows\Shell\BagsFenster-Parameter (Groesse, Ansicht, Spalten)
USRCLASS.DAT\Local Settings\Software\Microsoft\Windows\Shell\BagMRUPro-App-Container-Variante (Win8+)
Anti-Forensik-Hinweis: Shellbags persistieren auch nach Loeschung der Ordner und nach CCleaner-Durchlauf. Nur Tools wie BagsMRU_Cleaner.bat oder manuelles Loeschen der Registry-Keys entfernen sie. Sie sind eines der hartnaeckigsten Windows-Artefakte.

Tools: ShellBags Explorer (Zimmerman)

SRUM — System Resource Usage Monitor

AspektBeschreibung
FundortC:\Windows\System32\sru\SRUDB.dat (ESE-Datenbank)
Enthaltene Tabellen{973F5D5C-1D90-4944-BE8E-24B94231A174} = Network Data Usage
{D10CA2FE-6FCF-4F6D-848E-B2E99266FA89} = Application Resource Usage
{FEE4E14F-02A9-4550-B5CE-5FA2DA202E37} = Energy Usage
Forensische RelevanzPro Prozess: gesendete/empfangene Bytes, CPU-Zeit, Laufzeit. Bis zu 30-60 Tage Historie.
SRUM vs. Prefetch: Prefetch zeigt bis zu 8 letzte Ausfuehrungen. SRUM zeigt komplette Nutzungsdauer (wie viele Sekunden lief das Programm) und Netzwerkverkehr pro App. Beides zusammen = unschlagbare Kombination.

Tools: SrumECmd.exe (Zimmerman)

BAM und DAM — Background Activity Moderator

ArtefaktRegistry-KeyInhalt
BAM (v1709+)SYSTEM\CurrentControlSet\Services\bam\UserSettings\<SID>Prozessname + Pfad + letzter Ausfuehrungszeitpunkt (64-Bit FILETIME)
DAM (v1809+)SYSTEM\CurrentControlSet\Services\dam\UserSettings\<SID>Wie BAM, aber Desktop Activity-spezifisch
Forensischer Vorteil: BAM/DAM werden vom Kernel gepflegt (nicht von Usermode-Tools) und sind daher schwerer zu manipulieren als Prefetch. Sie enthalten den vollstaendigen Pfad der ausgefuehrten Datei und den exakten Zeitpunkt in UTC.

AmCache vs. ShimCache — Programmausfuehrungs-Historie

MerkmalAmCache (Application Compatibility Cache)ShimCache (AppCompatCache)
FundortC:\Windows\AppCompat\Programs\AmCache.hve (Registry Hive)SYSTEM\CurrentControlSet\Control\Session Manager\AppCompatCache
Enthaltene DatenPfad, Datei-Groesse, SHA1-Hash, Compile-Time, Publisher, Ausfuehrungszeit?Pfad, Last-Modified-Zeit, File-Size, 64-Bit-Flag
Wann aktualisiert?Beim naechsten Systemstart (nach Ausfuehrung)Direkt beim ersten Aufruf einer neuen .exe
PersistenceBleibt nach DeinstallationBleibt nach Loeschung der .exe
ToolsAmcacheParser.exe (Zimmerman)ShimCacheParser.py (Mandiant)

UserAssist — ROT13-Decodierung

# UserAssist speichert Programmnamen ROT13-codiert:
# chr(A-Z → N-ZA-M) für Großbuchstaben
# chr(a-z → n-za-m) für Kleinbuchstaben
# Zahlen und Symbole bleiben unveraendert

# Beispiel:
# CHQRE.PCZ → P U R E . P C M (ROT13-Abbildung) = CURE.PCM (noch falsch)
# Tatsaechlich: jedes Zeichen + 13 im Alphabet:
# {CEBFFVZ} → e v g v m E → "evgvmE" → umgekehrt?
# Gemeint ist: URNQ → H E A D? Nein, der Algorithmus ist:
# Jeder Buchstabe wird um 13 Positionen rotiert (ROT13):
# Von "SyntFcbby.pbz" zu "FlagsCool.com" ... tatsaechliche Beispiele
# {F7OPG6VQ} → Firefox, {6NQ7QNT7} → Notepad, etc.
# Encoding: Base85 (nicht ROT13!) unter Windows 10+
#
# Tools: UserAssistView (Nirsoft), Registry Explorer
Korrektur: Unter Windows 7/8 ist es tatsaechlich ROT13. Unter Windows 10+ verwendet UserAssist eine Base85-Encoding. Tools wie Registry Explorer (Zimmerman) decodieren automatisch.

Linux-Forensik Commands

# Login-Historie
last -F -f /var/log/wtmp           # Alle Logins mit vollem Zeitstempel
lastb -f /var/log/btmp             # Fehlgeschlagene Logins

# Systemd-Journal
journalctl --since "2025-01-01" --until "2025-01-02"
journalctl -u ssh.service          # Nur SSH-Logs
journalctl --list-boots            # Alle Boots mit IDs
journalctl -b -1                   # Vorheriger Boot

# Prozess-Historie (falls auditd aktiv)
ausearch -m EXECVE                 # Alle ausgefuehrten Programme
ausearch -m SYSCALL -sc execve     # execve-Syscalls

Praktische Anwendung

# Windows: Prefetch analysieren
PECmd.exe -d C:\Windows\Prefetch\ --csv C:\output\prefetch_out

# Windows: LNK analysieren
LECmd.exe -d "C:\Users\victim\AppData\Roaming\Microsoft\Windows\Recent\" --csv C:\output\lnk_out

# Windows: Jump Lists
JLECmd.exe -d "C:\Users\victim\AppData\Roaming\Microsoft\Windows\Recent\AutomaticDestinations\" --csv C:\output\jl_out

# Windows: SRUM exportieren
SrumECmd.exe -f C:\Windows\System32\sru\SRUDB.dat --csv C:\output\srum_out

# Windows: Shellbags aus Registry-Hive
# 1. NTUSER.DAT mit Registry Explorer laden
# 2. Navigiere zu: Software\Microsoft\Windows\Shell\BagMRU
# 3. ShellBags Explorer → File → Load NTUSER.DAT

# Windows: AmCache analysieren
AmcacheParser.exe -f C:\Windows\AppCompat\Programs\AmCache.hve --csv C:\output\amcache_out

# Linux: Alle verfuegbaren Artefakte sammeln
mkdir linux_forensics
cp ~/.bash_history linux_forensics/
cp /var/log/auth.log linux_forensics/
cp /var/log/wtmp linux_forensics/
journalctl --no-pager > linux_forensics/journal_full.txt
last -F > linux_forensics/last_output.txt
find /var/spool/cron/ -type f -exec cp {} linux_forensics/ \;

# macOS: FSEvents aus Image extrahieren
# (mac_apt kann FSEvents aus einem E01/dd/mount extrahieren)
mac_apt --input /mnt/image --output /cases/mac/output
