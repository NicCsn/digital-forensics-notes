Arbeitsspeicher-Forensik (Memory Forensics) Mittel

Inhaltsverzeichnis
  1. Kurzüberblick
  2. Details
  3. Praktische Anwendung
  4. Quellen / Weiterführendes

Kurzüberblick

Arbeitsspeicher (RAM) ist flüchtig — nach dem Ausschalten sind die Daten weg. Aber im laufenden Betrieb enthält er ALLES: Prozesse, Passwörter, Netzwerkverbindungen, Malware, Klartexte. Memory Forensics ist oft DER entscheidende Schritt, um fileless Malware oder laufende Angriffe zu identifizieren.

Details

RAM-Dump erstellen

Windows:

  • WinPMEM (Rekall-Toolkit), DumpIt, Magnet RAM Capture
  • Für VMs: Snapshot = RAM-Dump der VM

Linux:

  • LiME (Linux Memory Extractor): Kernel-Modul laden → RAM dumpen
  • insmod lime.ko path=/tmp/mem.dump format=lime
  • fmem (ältere Alternative)

macOS:

  • osxpmem, Recon Imager
  • macOS SIP erschwert Kernel-Module → SIP ggf. temporär deaktivieren

Volatility — Das Werkzeug

Volatility ist das Standard-Framework zur RAM-Analyse:

volatility -f mem.dump imageinfo

# Prozessliste
volatility -f mem.dump --profile=Win10x64 pslist
volatility -f mem.dump --profile=Win10x64 pstree    # Eltern-Kind-Beziehungen
volatility -f mem.dump --profile=Win10x64 psxview   # versteckte Prozesse finden

# Netzwerkverbindungen
volatility -f mem.dump --profile=Win10x64 netscan

# Geladene DLLs
volatility -f mem.dump --profile=Win10x64 dlllist

# Kommandozeilen der Prozesse
volatility -f mem.dump --profile=Win10x64 cmdline

# Injezierten Code finden
volatility -f mem.dump --profile=Win10x64 malfind

# Dateien aus dem RAM extrahieren
volatility -f mem.dump --profile=Win10x64 dumpfiles

# Registry-Hives aus dem RAM
volatility -f mem.dump --profile=Win10x64 hivelist

Typische Analyse-Ablauf

  • imageinfo → OS-Profil bestimmen (Win7? Win10? x64?)
  • pstree → Welche Prozesse laufen, welche Eltern-Kind-Verhältnisse?
  • cmdline → Wie wurde der Prozess gestartet?
  • netscan → Wohin bestehen Verbindungen?
  • malfind → Gibt es injizierten Code?
  • Suspicious Processes identifiziert? → memdump -p + Strings-Extraktion + YARA-Scan

Fileless Malware

  • Existiert nur im RAM, nie auf der Festplatte
  • Typische Vektoren: PowerShell-Empire, Cobalt Strike (Reflective DLL Loading)
  • Erkennung NUR per Memory Forensics möglich
  • Anzeichen: Prozess ohne zugehörige EXE-Datei, cmd.exe mit PowerShell-Kindprozess und Netzwerkverbindung

Praktische Anwendung

  • volatility3 (Python 3, aktiv entwickelt) vs. volatility2 (Python 2, Legacy)
  • strings mem.dump | grep -i password → Erste Schnellanalyse
  • vol -f mem.dump windows.cmdline.CmdLine → Volatility 3 Syntax

Quellen / Weiterführendes

  • Ligh, Case, Levy, Walters: The Art of Memory Forensics
  • Volatility Foundation

Emulatoren und erweiterte Volatility-Plugins

Code-Emulatoren für Memory-Analyse

Unicorn Engine

  • QEMU-basierte CPU-Emulator-Bibliothek, nutzbar via Python/C
  • Unterstützt: x86, x86-64, ARM, ARM64, MIPS, SPARC, m68k
  • Kein OS-Overhead — CPU-Instruktionen pur emulieren
  • Forensischer Nutzen: Shellcode aus RAM-Dump extrahieren und in Unicorn emulieren um Verhalten zu analysieren OHNE echte Malware-Umgebung
  • Beispiel-Workflow: 1) Volatility malfind → verdächtige Code-Region, 2) Hexdump → Unicorn laden, 3) API-Hooks setzen (kernel32.dll-Emulation), 4) Verhalten protokollieren
  • Python-API: uc_open(), uc_mem_map(), uc_mem_write(), uc_emu_start()

QEMU User-Mode

  • Einzelnes Binary (kein OS) in isolierter Umgebung ausführen
  • Syscall-Translation: Linux-Syscalls des Gasts werden auf Host-Syscalls übersetzt
  • Forensischer Nutzen: Malware-Binary aus Memory-Extract (via Volatility procdump) in QEMU user-mode ausführen

speakeasy (FireEye/Mandiant)

  • Windows-Kernel-API-Emulator, speziell für Malware-Analyse entwickelt
  • Emuliert kernel32.dll, ntdll.dll, ws2_32.dll etc.
  • Extrahiert API-Call-Sequenz, Netzwerk-Verbindungen, Dateisystem-Zugriffe — aus Emulation!

Volatility 3 Output-Formate

  • Standard: Text-Tabelle (--renderer text)
  • JSON: --renderer json → für automatisierte Analyse-Pipelines (Python, jq)
  • SQLite: --renderer sqlite → Datenbank für spätere Abfragen und Korrelationen

Praxisbeispiel: Komplette Memory-Analyse mit Volatility 3

# 1. Image-Info
vol -f memdump.raw windows.info

# 2. Prozessliste + versteckte Prozesse
vol -f memdump.raw windows.pslist
vol -f memdump.raw windows.psscan

# 3. Commandlines — was wurde gestartet?
vol -f memdump.raw windows.cmdline

# 4. Netzwerkverbindungen
vol -f memdump.raw windows.netstat
vol -f memdump.raw windows.netscan

# 5. Malware-Scan
vol -f memdump.raw windows.malfind

# 6. Injection-Indikatoren (Mutexes/Handles)
vol -f memdump.raw windows.handles --pid 1234 | grep -i mutant

# 7. Verdächtiges Binary extrahieren
vol -f memdump.raw windows.dumpfiles --pid 1234 --virtaddr 0x400000

# 8. Registry-Auswertung
vol -f memdump.raw windows.registry.hivelist
vol -f memdump.raw windows.registry.userassist

# 9. Credential-Dump
vol -f memdump.raw windows.hashdump
