Kurzueberblick

MITRE ATT&CK (Adversarial Tactics, Techniques, and Common Knowledge) ist das weltweit genutzte Framework zur Beschreibung von Angreiferverhalten. Es katalogisiert reale TTPs (Tactics, Techniques, Procedures), die von APT-Gruppen und Cyberkriminellen eingesetzt werden. Fuer die Digitale Forensik ist ATT&CK das zentrale Werkzeug, um Spuren anhand der Angreifer-Methodik zu erkennen und zu klassifizieren.

Version: ATT&CK v16 (Januar 2026). Es gibt Matrizen fuer Enterprise (Windows, macOS, Linux, Cloud, Network, Container), Mobile (Android, iOS) und ICS.

Matrix-Struktur: Tactics x Techniques

Die ATT&CK-Matrix organisiert Angriffsverhalten in zwei Dimensionen: Tactics (Taktiken, Spalten) — das WAS (Ziel des Angreifers) — und Techniques (Techniken, Zellen) — das WIE (konkrete Methode). Jede Technik kann Sub-Techniques haben, die Varianten oder spezifische Implementierungen beschreiben.

Techniques und Sub-Techniques

Jede Technik hat eine ID im Format TXXXX (z.B. T1059 = Command and Scripting Interpreter). Sub-Techniques erweitern: T1059.001 = PowerShell, T1059.003 = Windows Command Shell.

Beispiel: T1059 — Command and Scripting Interpreter

IDSub-TechniqueErkennung (Sysmon/EventLog)
T1059.001PowerShellEventID 4104 (Script Block Logging), 400 (Engine-Start)
T1059.003Windows Command ShellEventID 4688 (Process Creation) mit cmd.exe
T1059.005Visual Basiccscript.exe, wscript.exe, Office-Makros
T1059.007JavaScriptjscript.dll, mshta.exe
Technique vs Procedure: T1059.001 ist die TECHNIK (PowerShell). Die PROCEDURE ist, WIE eine bestimmte APT-Gruppe PowerShell nutzt — z.B. "APT29 nutzt PowerShell mit Base64-codierten Commands und -WindowStyle Hidden". Procedures findest du in ATT&CK Groups und Threat-Intelligence-Reports.

ATT&CK Navigator

Der ATT&CK Navigator ist eine interaktive Web-App zur Visualisierung. Du erstellst Layers, die die Matrix farblich kodieren:

  • Gruen = Erkannt: Du hast Logs/Detection fuer diese Technik
  • Gelb = Teilweise: Du kannst diese Technik bedingt erkennen
  • Rot = Blind Spot: Keine Erkennung fuer diese Technik

Navigator Layers koennen als JSON exportiert und mit anderen Teams geteilt werden. Forensischer Use-Case: Erstelle einen Layer, der zeigt, welche Techniken du in einem konkreten Incident NACHWEISEN konntest. Gruen = Beweis gefunden.

Log-Quellen auf ATT&CK mappen

Der forensische Mehrwert von ATT&CK liegt im Mapping von Log-Quellen zu Techniken. Jede Technik-Detailseite auf attack.mitre.org listet "Data Sources", die zur Erkennung genutzt werden koennen:

Detection Gap Analysis

Erstelle eine ATT&CK-Navigator-Layer fuer deine Org:

  1. Markiere ALLE Techniken, die du erkennen KANNST (Log-Quellen vorhanden + Detection Rules)
  2. Markiere Techniken, die fuer DEINE Branche/threat-model relevant sind
  3. Die Schnittmenge "relevant aber nicht erkennbar" = Detection Gap
  4. Priorisiere Schliessung dieser Gaps nach Risiko (Kritikalitaet der Technik × Wahrscheinlichkeit)

Praktische Anwendung

  • Incident-Analyse: Welche ATT&CK-Taktiken wurden eingesetzt? Initial Access (T1566 Phishing) → Execution (T1059.001 PowerShell) → Persistence (T1547.001 Registry Run) → C2 (T1071.001 HTTP)
  • Reporting: "Der Angreifer nutzte T1059.001 (PowerShell) fuer lateral movement und T1003.001 (LSASS-Dump) fuer Credential Access" — standardisierte, weltweit verstaendliche Sprache
  • Sigma-Regeln schreiben: Tags in Sigma-Regeln enthalten oft ATT&CK-IDs: tags: attack.t1059, attack.t1059.001
  • Threat Intelligence: ATT&CK Groups (z.B. APT29, FIN7) zeigen, welche Techniken bestimmte Akteure bevorzugen
ATT&CK ist KEIN Audit-Checkliste: Nicht alle Techniken sind fuer jedes Unternehmen relevant. Ein KMU ohne Kubernetes braucht kein Container-Escape zu detektieren. Immer threat-model-basiert priorisieren.
