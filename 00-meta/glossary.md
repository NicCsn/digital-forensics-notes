# Glossar

Zentrale Begriffsammlung zur Digitalen Forensik, IT-Security und Informatik-Grundlagen.
Alphabetisch sortiert. Jeder Begriff verlinkt auf die zugehoerige Notiz im Repo.

---

| # | Begriff | Definition | siehe auch |
|---|---------|------------|------------|
| 1 | 2er-Komplement | Binaere Darstellung negativer Zahlen. Das hoechstwertige Bit ist das Vorzeichenbit. -1 = alle Bits auf 1. Bereich bei n Bit: -2^(n-1) bis 2^(n-1)-1 | [Zahlensysteme](01-grundlagen/zahlensysteme-und-kodierung.md) |
| 2 | 4-Way-Handshake (WPA) | Vier Nachrichten zwischen Client und AP zum Aushandeln des Pairwise-Transient-Key (PTK). Capture dieses Handshakes ermoeglicht Offline-Brute-Force des WLAN-Passworts | [Wireless-Forensik](03-netzwerke/wireless-forensik.md) |
| 3 | A-Record (DNS) | DNS-Record-Typ: bildet einen Hostnamen auf eine IPv4-Adresse ab. `www.example.com A 93.184.216.34` | [DHCP, DNS, ARP](03-netzwerke/dhcp-dns-arp.md) |
| 4 | AAAA-Record (DNS) | DNS-Record-Typ: bildet einen Hostnamen auf eine IPv6-Adresse ab | [DHCP, DNS, ARP](03-netzwerke/dhcp-dns-arp.md) |
| 5 | ABAC | Attribute-Based Access Control. Zugriffskontrolle basierend auf Attributen (Nutzer, Ressource, Umgebung) statt nur Rollen. Flexibler als RBAC | [Authentifizierung & PKI](05-it-security/authentifizierung-pki.md) |
| 6 | ACPO | Association of Chief Police Officers (UK). Veroeffentlichte die 4 Grundprinzipien fuer den Umgang mit digitalen Beweismitteln | [Forensischer Prozess](04-forensik-methoden/forensischer-prozess-chain-of-custody.md) |
| 7 | ADAM | Advanced Data Acquisition Model. Erweitert ACPO-Prinzipien um Sicherheit von Personen und Geraeten | [Forensischer Prozess](04-forensik-methoden/forensischer-prozess-chain-of-custody.md) |
| 8 | ADS (Alternate Data Streams) | NTFS-Feature: Dateien koennen unsichtbare Zusatzdatenstroeme enthalten. Wird von Malware als Versteck genutzt. Erkennbar mit `dir /R` oder `streams.exe` | [Anti-Forensik](02-betriebssysteme/anti-forensik-betriebssystem.md) |
| 9 | AES | Advanced Encryption Standard. Symmetrisches Blockverschluesselungsverfahren (128/192/256 Bit Schluessel). NIST-Standard seit 2001. Betriebsmodi: ECB (unsicher), CBC, CTR, GCM (authenticated) | [Kryptografie](05-it-security/kryptografie-grundlagen-symmetrisch-asymmetrisch.md) |
| 10 | ALU | Arithmetic Logic Unit. Teil der CPU, der arithmetische (Addition, Subtraktion) und logische (AND, OR, NOT) Operationen ausfuehrt | [Rechnerarchitektur](01-grundlagen/rechnerarchitektur-von-neumann.md) |
| 11 | AmCache | Eigenstaendige Registry-Hive-Datei (`Amcache.hve` unter `%SystemRoot%\AppCompat\Programs\`). Protokolliert ausgefuehrte Programme mit SHA1-Hash und Pfad. Nicht zu verwechseln mit ShimCache (im SYSTEM-Hive) | [Registry-Forensik](02-betriebssysteme/windows-registry-forensik.md) |
| 12 | AND-Gatter | Logisches Gatter: Ausgang ist 1 nur wenn ALLE Eingaenge 1 sind. A AND B = 1 nur bei A=1, B=1. Basis der Digitaltechnik | [Zahlensysteme](01-grundlagen/zahlensysteme-und-kodierung.md) |
| 13 | Anomalieerkennung | Identifikation von Daten, die nicht dem erwarteten Muster entsprechen (z.B. ungewoehnlicher Netzwerk-Traffic, abweichendes Benutzerverhalten) | [Netzwerkforensik](03-netzwerke/netzwerkforensik-methoden.md), [SIEM](05-it-security/siem-und-monitoring.md) |
| 14 | Anti-Forensik | Techniken zur Verschleierung oder Vernichtung digitaler Spuren. Dazu gehoeren Timestomping, Log-Wiping, sicheres Loeschen, Verschluesselung und Steganografie | [Anti-Forensik](02-betriebssysteme/anti-forensik-betriebssystem.md) |
| 15 | Anycast | Netzwerkadressierungsmethode: eine IP wird von mehreren Servern an verschiedenen Orten bedient. Routing leitet zum naechstgelegenen Server. Wird fuer DNS (8.8.8.8) und CDNs genutzt | [IP-Adressierung](03-netzwerke/ip-adressierung-subnetting.md) |
| 16 | APFS | Apple File System. Copy-on-Write, Snapshot-faehig, Space Sharing, native Verschluesselung. Standard unter macOS, iOS, watchOS, tvOS | [Dateisysteme](02-betriebssysteme/dateisysteme-uebersicht-fat-ntfs-ext4-apfs.md) |
| 17 | API | Application Programming Interface. Definiert, wie Software-Komponenten miteinander kommunizieren. REST, SOAP und GraphQL sind gaengige Web-API-Paradigmen | [Python-Grundlagen](01-grundlagen/python-grundlagen.md) |
| 18 | APIPA | Automatic Private IP Addressing (169.254.x.x). Wird von einem Geraet selbst zugewiesen, wenn kein DHCP-Server antwortet. Im Log ein Hinweis auf Netzwerkprobleme | [IP-Adressierung](03-netzwerke/ip-adressierung-subnetting.md) |
| 19 | ARP-Spoofing | Angriff: Angreifer sendet gefaelschte ARP-Antworten und leitet Traffic um (Man-in-the-Middle). Erkennbar an mehreren MAC-Adressen fuer dieselbe IP in der ARP-Tabelle | [DHCP, DNS, ARP](03-netzwerke/dhcp-dns-arp.md) |
| 20 | ASCII | American Standard Code for Information Interchange. 7-Bit-Zeichenkodierung (128 Zeichen: Buchstaben, Ziffern, Steuerzeichen). Urvater von UTF-8 und Latin-1 | [Zahlensysteme](01-grundlagen/zahlensysteme-und-kodierung.md) |
| 21 | ASLR | Address Space Layout Randomization. Sicherheitsmechanismus: Position von Stack, Heap und Bibliotheken im Speicher wird zufaellig gewaehlt. Erschwert Buffer-Overflow-Exploits, da Angreifer keine festen Adressen hat | [Anti-Forensik](02-betriebssysteme/anti-forensik-betriebssystem.md) |
| 22 | Assembler | Uebersetzt menschenlesbare Mnemonics (MOV, ADD, JMP) in Maschinencode. Jede CPU-Architektur hat ihren eigenen Befehlssatz (x86, ARM, RISC-V) | [Rechnerarchitektur](01-grundlagen/rechnerarchitektur-von-neumann.md) |
| 23 | Authentifizierung | Nachweis einer Identitaet. Drei Faktoren: Wissen (Passwort), Besitz (Token), Inhaerenz (Biometrie). MFA kombiniert >=2 Faktoren | [Authentifizierung & PKI](05-it-security/authentifizierung-pki.md) |
| 24 | Autopsy | Open-Source-Forensik-Plattform (GUI fuer Sleuth Kit). Bietet Case-Management, Timeline-Analyse, Keyword-Suche, Hash-Filtering und integrierte Reporting-Funktionen | [Datentraeger-Forensik](04-forensik-methoden/datentraeger-forensik.md) |
| 25 | BAM (Background Activity Moderator) | Windows-Registry-Key unter `HKLM\SYSTEM\CurrentControlSet\Services\bam\State\UserSettings`. Protokolliert letzte Ausfuehrungszeit jeder ausgefuehrten Datei pro Benutzer | [Spurensuche](02-betriebssysteme/betriebssystem-forensik-spurensuche.md) |
| 26 | Base64 | Kodierung, die Binaerdaten in 64 druckbare ASCII-Zeichen (A-Z, a-z, 0-9, +, /) umwandelt. 3 Bytes -> 4 Zeichen. Padding mit = oder ==. Wird in E-Mails (MIME) und JSON/XML genutzt | [Zahlensysteme](01-grundlagen/zahlensysteme-und-kodierung.md) |
| 27 | Beaconing | Regelmaessige, intervallgetreue Verbindungen von Malware zu einem C2-Server. Z.B. alle 60 Sekunden ein HTTP-Request. Erkennbar in PCAPs und NetFlow-Daten | [Netzwerkforensik](03-netzwerke/netzwerkforensik-methoden.md) |
| 28 | Beschlagnahme | Strafprozessuale Sicherstellung eines Gegenstandes. Im Unterschied zur Durchsuchung: Beschlagnahme = Gegenstand wird in amtliche Verwahrung genommen | [Digitale Beweismittel](07-recht-kriminalistik/digitale-beweismittel-grundprinzipien.md) |
| 29 | Best Evidence Rule | Grundsatz: Das beste verfuegbare Original muss als Beweis vorgelegt werden. Im digitalen Kontext wird ein forensisch korrektes Abbild (Image) als Original anerkannt | [Digitale Beweismittel](07-recht-kriminalistik/digitale-beweismittel-grundprinzipien.md) |
| 30 | Betriebssystem | Software, die Hardware-Ressourcen verwaltet und Anwendungen eine standardisierte Plattform bietet (Prozess-Scheduling, Speicherverwaltung, Dateisystem, Treiber) | [BS-Architekturen](02-betriebssysteme/betriebssystem-architekturen.md) |
| 31 | BGP (Border Gateway Protocol) | Das Routing-Protokoll des Internets. Tauscht Erreichbarkeitsinformationen zwischen autonomen Systemen (AS) aus. BGP-Hijacking: boeswillige Umleitung von IP-Praefixen | [Routing & Switching](03-netzwerke/routing-switching-vlans.md) |
| 32 | BinDatei / Binary | Von-Neumann-Architektur: Computer speichert alles binaer. 0 oder 1 = ein Bit. 8 Bit = 1 Byte. Basis aller Kodierung | [Zahlensysteme](01-grundlagen/zahlensysteme-und-kodierung.md) |
| 33 | Bit | Kleinste Informationseinheit: 0 oder 1. 8 Bit = 1 Byte. n Bit koennen 2^n verschiedene Werte darstellen | [Zahlensysteme](01-grundlagen/zahlensysteme-und-kodierung.md) |
| 34 | BitLocker | Microsofts Vollverschluesselungsloesung fuer Windows (AES 128/256 Bit). Schluesselverwaltung ueber TPM, USB-Stick, PIN oder Recovery-Key (48-stellig). RAM-Dump bei laufendem System kann Schluessel enthalten | [Datentraeger-Verschluesselung](02-betriebssysteme/datentraeger-verschluesselung.md) |
| 35 | Blue Team | Defensives Sicherheitsteam. Aufgaben: Monitoring, Incident Response, Threat Hunting, Haertung. Gegenstueck zum Red Team (offensiv) und ergaenzt durch Purple Team (Zusammenarbeit) | [Incident Management](05-it-security/security-incident-management.md) |
| 36 | Boolesche Algebra | Mathematisches System mit den Werten 0 und 1 und den Operatoren AND, OR, NOT. Grundlage der Digitaltechnik und Schaltalgebra | [Zahlensysteme](01-grundlagen/zahlensysteme-und-kodierung.md) |
| 37 | Bootloader | Programm, das beim Start den Betriebssystem-Kernel in den Speicher laedt. GRUB (Linux), Windows Boot Manager. UEFI Secure Boot verifiziert die Signatur des Bootloaders | [BS-Architekturen](02-betriebssysteme/betriebssystem-architekturen.md) |
| 38 | Broadcast | Netzwerkadressierung an ALLE Geraete im Netzsegment. IPv4: letzte Adresse im Subnetz (z.B. 192.168.1.255/24). MAC: FF:FF:FF:FF:FF:FF. Wird fuer ARP und DHCP-Discovery genutzt | [IP-Adressierung](03-netzwerke/ip-adressierung-subnetting.md) |
| 39 | Brute-Force | Systematisches Durchprobieren aller moeglichen Kombinationen (Passwoerter oder Schluessel). Aufwand steigt exponentiell mit Laenge und Zeichensatz | [Kryptoanalyse](05-it-security/kryptoanalyse.md) |
| 40 | Byte | 8 Bit. Auf den meisten Architekturen die kleinste adressierbare Speichereinheit. Wertebereich 0-255 (unsigned) oder -128 bis 127 (signed, 2er-Komplement) | [Zahlensysteme](01-grundlagen/zahlensysteme-und-kodierung.md) |
| 41 | Bytecode | Zwischencode zwischen Quellcode und Maschinencode. Wird von einer virtuellen Maschine ausgefuehrt (JVM, CPython, .NET CLR). Plattformunabhaengig | [Programmierparadigmen](01-grundlagen/programmierparadigmen.md) |
| 42 | C2 (Command & Control) | Infrastruktur, ueber die Angreifer kompromittierte Systeme steuern. Kommunikation oft ueber HTTP(S), DNS oder social media. Erkennung durch Beaconing-Analyse | [Malware dynamisch](04-forensik-methoden/malware-analyse-dynamisch.md) |
| 43 | CaaS | Cybercrime-as-a-Service. Kriminelle Dienstleistungen als Abo-Modell: Malware-Entwicklung, Bulletproof-Hosting, Initial-Access-Brokering, Money-Muling | [Cybercrime-Oekonomie](06-osint/cybercrime-oekonomie.md) |
| 44 | Cache | Kleiner, schneller Zwischenspeicher zwischen CPU und RAM (L1/L2/L3). Enthaelt kuerzlich verwendete Daten. Forensisch: Cache kann fluechtige Spuren enthalten, die nie auf die Platte geschrieben wurden | [Rechnerarchitektur](01-grundlagen/rechnerarchitektur-von-neumann.md) |
| 45 | Caesar-Chiffre | Historische monoalphabetische Substitution: jeder Buchstabe wird um k Positionen im Alphabet verschoben. Leicht durch Haeufigkeitsanalyse zu brechen | [Kryptografie](05-it-security/kryptografie-grundlagen-symmetrisch-asymmetrisch.md) |
| 46 | CDN | Content Delivery Network. Verteilt Inhalte ueber weltweit verteilte Server. Forensisch: CDN-Logs koennen Zugriffszeitpunkte und IPs des Nutzers enthalten (z.B. Cloudflare, Akamai) | [Cloud-Forensik](04-forensik-methoden/cloud-forensik.md) |
| 47 | Chain of Custody | Lueckenlose Dokumentation aller Zugriffe und Uebergaben eines Beweismittels (Wer? Wann? Was? Warum?). Voraussetzung fuer die gerichtliche Zulassung | [Forensischer Prozess](04-forensik-methoden/forensischer-prozess-chain-of-custody.md) |
| 48 | CIA-Triade | Drei Schutzziele der Informationssicherheit: Confidentiality (Vertraulichkeit), Integrity (Integritaet), Availability (Verfuegbarkeit). Erweitert um Authentizitaet und Nicht-Abstreitbarkeit | [Schutzziele (CIA)](05-it-security/schutzziele-cia-triad.md) |
| 49 | CIDR | Classless Inter-Domain Routing. Notation fuer IP-Netzbereiche: `192.168.1.0/24`. /24 = 255.255.255.0 = 254 nutzbare Hosts | [IP-Adressierung](03-netzwerke/ip-adressierung-subnetting.md) |
| 50 | CNAME-Record (DNS) | Canonical Name. DNS-Record-Typ: Alias von einem Hostnamen auf einen anderen. `www.example.com CNAME example.com` | [DHCP, DNS, ARP](03-netzwerke/dhcp-dns-arp.md) |
| 51 | CNN | Convolutional Neural Network. Deep-Learning-Architektur spezialisiert auf raeumliche Strukturen (Bilder). Filter (Kernel) gleiten ueber das Eingabebild | [Neuronale Netze](08-ki-forensik/neuronale-netze-deep-learning.md) |
| 52 | Compiler | Uebersetzt Quellcode in Maschinencode oder Bytecode VOR der Ausfuehrung (AOT = Ahead-of-Time). Im Unterschied zum Interpreter, der zur Laufzeit uebersetzt | [Programmierparadigmen](01-grundlagen/programmierparadigmen.md) |
| 53 | CRAAP-Test | Quellenbewertung im OSINT: Currency (Aktualitaet), Relevance (Relevanz), Authority (Autoritaet), Accuracy (Genauigkeit), Purpose (Zweck) | [OSINT-Recherche](06-osint/osint-recherche-methodik.md) |
| 54 | CSRF | Cross-Site Request Forgery. Angreifer bringt Browser des Opfers dazu, ungewollte Aktionen auf einer Webanwendung auszufuehren (z.B. Geld ueberweisen). Schutz durch CSRF-Token | [Web-Sicherheit](03-netzwerke/web-sicherheit-owasp.md) |
| 55 | CVE / CVSS | Common Vulnerabilities and Exposures (Eindeutige ID fuer Schwachstellen: CVE-2023-XXXXX) und Common Vulnerability Scoring System (Schweregrad 0-10) | [Web-Sicherheit](03-netzwerke/web-sicherheit-owasp.md) |
| 56 | CWE | Common Weakness Enumeration. Hierarchische Liste von Software-Schwachstellen-Typen. CWE-79 = XSS, CWE-89 = SQL Injection | [Web-Sicherheit](03-netzwerke/web-sicherheit-owasp.md) |
| 57 | Daemon | Hintergrundprozess unter Unix/Linux. Wird beim Booten gestartet, laeuft ohne Benutzerinteraktion. z.B. sshd, httpd, cron. Windows-Aequivalent: Service | [Linux-Administration](02-betriebssysteme/linux-administration.md) |
| 58 | DAM (Desktop Activity Moderator) | Windows-Registry-Key: aehnlich BAM, aber in `HKLM\SYSTEM\CurrentControlSet\Services\dam\`. Weniger dokumentiert, gleiche Funktion | [Spurensuche](02-betriebssysteme/betriebssystem-forensik-spurensuche.md) |
| 59 | Darknet | Ueber Tor oder I2P zugaenglicher, nicht durch Suchmaschinen indexierter Teil des Internets. Genutzt fuer anonyme Kommunikation, Whistleblowing und illegale Marktplaetze | [Darknet & Tor](03-netzwerke/darknet-tor-i2p.md) |
| 60 | dd | Bit-genaues Kopierwerkzeug unter Unix/Linux. `dd if=/dev/sda of=image.dd`. Forensische Variante: `dcfldd` mit integriertem Hashing | [Forensisches Imaging](04-forensik-methoden/forensisches-imaging.md) |
| 61 | Deepfake | KI-generierte synthetische Medien (Gesichter oder Stimmen). Erkennung durch CNNs, die subtile Artefakte identifizieren (unregelmaessiges Blinzeln, inkonsistente Beleuchtung) | [Bildforensik](08-ki-forensik/bildforensik-objekterkennung.md) |
| 62 | DEP / NX-Bit | Data Execution Prevention. Sicherheitsmechanismus: Speicherbereiche werden als "nicht ausfuehrbar" markiert. Verhindert Code-Ausfuehrung auf Stack/Heap. NX = AMD, XD = Intel | [Memory Management](02-betriebssysteme/memory-management.md) |
| 63 | Design Pattern | Wiederverwendbare Loesungsschablone fuer haeufige Software-Entwurfsprobleme. Singleton (eine Instanz), Factory (Erzeugung), Observer (Ereignisbenachrichtigung) | [OOP](01-grundlagen/oop-klassen-vererbung.md) |
