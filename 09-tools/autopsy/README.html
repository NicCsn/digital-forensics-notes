Kurzueberblick

Autopsy ist eine grafische Open-Source-Forensik-Plattform, die auf dem Sleuth Kit (TSK) aufbaut. Sie ermoeglicht die Analyse von Festplatten-Images (E01, dd/raw, AFF), lokalen Laufwerken und logischen Dateisammlungen. Autopsy ist die GUI — das Sleuth Kit ist die darunterliegende Engine, die Dateisystem-Metadaten extrahiert.

EigenschaftBeschreibung
EntwicklerBasis Technology / Community
LizenzApache 2.0 (Open Source)
PlattformWindows, Linux, macOS
BasisSleuth Kit (TSK) + integrierte Tools
Image-Formatedd/raw, E01 (EWF), AFF, VMDK, VHD
DatenbankSQLite (Case-Datenbank)
Autopsy vs. Sleuth Kit: Das Sleuth Kit (TSK) ist die Commandline-Engine fuer Dateisystem-Analyse (fls, fsstat, mmls, icat, etc.). Autopsy ist die grafische Oberflaeche, die TSK + Keyword-Suche + Timeline + Hash-Datenbank + Reporting in einer Anwendung vereint.

Case-Erstellung

Jede Analyse in Autopsy beginnt mit einem Case. Ein Case buendelt alle Datenquellen, Analyseergebnisse und Reports in einer SQLite-Datenbank.

# Case-Erstellung (GUI-Workflow)
1. "New Case" klicken
2. Case Name:     z.B. "2026-07-02_Mustermann_HDD"
3. Base Directory: z.B. D:\Cases\  (hier werden alle Case-Daten gespeichert)
4. Optional: Case-Nummer, Ermittler-Name, Organisation
5. "Finish" → Case ist erstellt
FeldBedeutungForensische Relevanz
Case NameEindeutiger Name fuer diesen FallKonvention: Datum_Nachname_Medium (z.B. 2026-07-02_Schmidt_Laptop)
Base DirectoryOrdner, in dem Case-DB und Reports gespeichert werdenAuf schnellem Speicher ablegen (SSD), NICHT auf dem Beweismittel-Laufwerk!
Case NumberInterne FallnummerOptional, aber hilfreich fuer Chain of Custody
ExaminerName des UntersuchersWird in Reports uebernommen, wichtig fuer Dokumentation
Case-Verzeichnis: Autopsy erstellt im Base Directory einen Unterordner mit dem Case-Namen. Dieses Verzeichnis enthaelt die SQLite-DB und spaeter Reports. Nie auf dem Quell-Image speichern — immer auf einem separaten Analyse-Laufwerk.

Evidence hinzufuegen

DatenquelleBeschreibungWann verwenden?
Disk Image or VM FileForensisches Image (E01, dd, AFF, VMDK)Standardfall: Image wurde mit FTK Imager / Guymager / dd erstellt
Local DrivePhysisches Laufwerk oder Partition des Analyse-ComputersLive-Analyse (nur mit Write-Blocker!)
Logical FilesEinzelne Dateien oder OrdnerLogfile-Analyse, Email-PST, Browser-Cache-Ordner
# Image hinzufuegen (Workflow)
1. "Add Data Source" klicken
2. "Disk Image or VM File" auswaehlen
3. Pfad zum Image angeben (z.B. D:\Images\Verdaechtiger.E01)
4. "Symantec" = Symlink zu Image (Standard, empfohlen)
5. "Copy" = Image in Case-Verzeichnis kopieren (nur bei kleinen Images!)
6. Timezone auswaehlen (fuer korrekte Timestamps)
7. Ingest-Module auswaehlen (s. naechster Abschnitt)
Ingest-Module beim Hinzufuegen: Autopsy fragt direkt beim Hinzufuegen von Evidence, welche Ingest-Module ausgefuehrt werden sollen. Standardmaessig laufen die wichtigsten (File Type, Hash, EXIF), aber bei grossen Images besser selektiv aktivieren, um Zeit zu sparen.

Ingest-Module

Ingest-Module verarbeiten die Rohdaten des Images und extrahieren zusaetzliche Informationen. Sie laufen nach dem Hinzufuegen der Evidence automatisch durch.

ModulFunktionOutput
Recent ActivityExtrahiert kuerzlich genutzte Dateien, Web-Verlauf, kuerzlich geoeffnete DocsTimeline der Benutzeraktivitaet
Hash LookupBerechnet Hashes (MD5/SHA) und gleicht mit NSRL (known-good) abMarkiert "known bad" (auffaellig) und "known good" (Betriebssystem)
File Type IDErmittelt Dateityp via Magic Bytes, nicht nur via ExtensionFindet versteckte Dateien (z.B. .jpg ist eigentlich .exe)
Extension MismatchVergleicht Extension mit tatsaechlichem DateitypListe aller Dateien mit falscher Endung
EXIF ParserExtrahiert Metadaten aus JPEG, TIFF, etc.Kamera-Modell, GPS-Koordinaten, Aufnahmezeit
Keyword SearchIndexiert Text-Inhalte fuer VolltextsucheSuchindex (Regex und Literal)
Email ParserPST, OST, MBOX parsenEmails, Kontakte, Kalender
Embedded File ExtractorExtrahiert Dateien aus Archives (ZIP, 7z, RAR), Office-Docs, PDFsEmbedded files als separate Artefakte
Interesting FilesSucht nach Patterns wie Kreditkartennummern, IBAN, URLs, Email-AdressenListe mit Fundstellen
PlasoErstellt eine Super-Timeline (alle Timestamps aus allen Quellen)Chronologische Timeline aller Events
Performance-Warnung: Keyword Search und Plaso koennen bei grossen Images (500 GB+) viele Stunden oder Tage benoetigen. Bei Triage-Situationen selektiv deaktivieren oder auf verdaechtige Ordner beschraenken.

Analyse-Werkzeuge

File Browser (Datei-Explorer)

Der File Browser zeigt die Dateisystem-Hierarchie aus Sicht des Betriebssystems, inklusive geloeschter Dateien (als "Deleted" markiert). Er ist der Haupt-Einstiegspunkt fuer die Navigation im Image.

  • File Views: File Types (nach Typ gruppiert), Deleted Files (nur geloeschte), File Size (Top-100 groesste)
  • Views by Data Source: Alle Dateien aus einer bestimmten Quelle (bei mehreren Images in einem Case)
  • Hex Viewer: Integrierter Hex-Viewer fuer einzelne Dateien

Keyword Search

Autopsy unterstuetzt zwei Suchmodi:

SuchmodusBeschreibungBeispiel
Literal SearchExakte Zeichenkette (Case-Insensitive optional)password, confidential, bitcoin
Regex SearchRegulaere Ausdruecke\b[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Z|a-z]{2,}\b (Email-Adressen)
Index-Suche: Wenn das Keyword-Search-Ingest-Modul beim Import aktiviert wurde, ist die Suche instant. Ohne Index durchsucht Autopsy die Dateien live — das kann bei grossen Images sehr langsam sein.

Timeline

Die Timeline-Ansicht zeigt alle Datei-Ereignisse chronologisch. Sie kann nach Datum (Kalender) oder als lineare Liste gefiltert werden. Besonders wertvoll: Zoom auf den Zeitraum des Vorfalls, dann analysieren, welche Dateien erstellt, modifiziert oder geloescht wurden.

Communications

Die Communications-Ansicht aggregiert Kommunikationsdaten aus verschiedenen Quellen:

  • Emails: PST/OST/MBOX-Parsing, sortiert nach Absender, Empfaenger, Datum
  • Chat: Skype, Slack, WhatsApp (aus Mobile-Extractions)
  • Web: Browser-History (Chrome, Firefox, Edge), Downloads, Cookies

Data Artifacts

Data Artifacts fassen Ergebnisse der Ingest-Module in strukturierten Listen zusammen:

  • Web Bookmarks, Cookies, History, Downloads
  • Recent Documents (Windows Recent, macOS Recent Items)
  • USB Devices: Angeschlossene USB-Geraete mit Seriennummern
  • Installed Programs: Registry-basierte Software-Liste
  • Operating System Info: OS-Version, Hostname, Installationsdatum
  • Shellbags: Explorer-Ordnerzugriffe

Reporting

Autopsy bietet zwei Export-Formate fuer Reports:

Report-TypInhaltFormat
HTML ReportVollstaendiger Bericht mit Tabellen, Thumbnails, File DetailsHTML + Assets (CSS, Bilder)
Excel ExportTabellen-Export fuer Excel mit Filter-OptionenXLSX
# Report erstellen
1. "Generate Report" klicken
2. Report-Typ auswaehlen (HTML oder Excel)
3. Datenquellen auswaehlen (welche Images sollen enthalten sein?)
4. Modul-Ergebnisse auswaehlen (alle oder selektiv)
5. "Finish" — Report wird im Case-Verzeichnis gespeichert
Report als Arbeitsgrundlage: Der HTML-Report wird oft als strukturierte Uebersicht verwendet, aus der dann ein detaillierter, gerichtsfester Bericht in Word/LaTeX geschrieben wird. Der Autopsy-Report enthaelt bereits Screenshots, Timelines und Fundort-Referenzen.

Praktische Anwendung: Imaging importieren → Ingest → Analyse

Szenario: Du hast mit FTK Imager ein E01-Image einer verdaechtigen 500 GB HDD erstellt. Jetzt soll das Image in Autopsy analysiert werden.

  1. Autopsy starten und "New Case" anlegen:
    • Case Name: 2026-07-02_Verdaechtiger_HDD
    • Base Directory: D:\Cases
  2. Data Source hinzufuegen: "Disk Image or VM File" → D:\Images\Verdaechtiger.E01 auswaehlen. Timezone auf "Europe/Berlin" setzen.
  3. Ingest-Module konfigurieren (empfohlene Auswahl):
    • ✅ Recent Activity
    • ✅ Hash Lookup (NSRL-DB vorher herunterladen!)
    • ✅ File Type ID
    • ✅ Extension Mismatch
    • ✅ EXIF Parser
    • ❌ Keyword Search (erst spaeter aktivieren oder auf verdaechtige Ordner einschraenken)
    • ✅ Email Parser (nur wenn PST/OST vorhanden)
  4. Ingest laeuft — abwarten, bis alle Module durch sind.
  5. Erste Analyse-Schritte:
    • "Data Artifacts" → "Recent Activity" checken: Welche Dateien wurden zuletzt verwendet?
    • "File Views" → "Deleted Files": Welche Dateien wurden geloescht?
    • "File Views" → "File Types" → "Executable": Unbekannte EXE-Dateien identifizieren
    • "Extension Mismatch": Dateien suchen, deren Endung nicht zum Typ passt
  6. Verdaechtige Dateien prufen: Hex-Viewer oeffnen → Strings analysieren → Hash auf VirusTotal ueberpruefen
  7. Report generieren → HTML-Report als Grundlage fuer die Dokumentation
Write-Blocker nicht vergessen: Wenn du ein physisches Laufwerk ("Local Drive") an deinen Analyse-Rechner anschliesst, MUSS ein Hardware-Write-Blocker zwischen Laufwerk und Computer geschaltet sein. Ohne Write-Blocker koennen Metadaten durch das Betriebssystem veraendert werden — das Beweismittel ist dann moeglicherweise unbrauchbar.
