Apache Log Parser Grundlagen

Was macht das Tool?

Der Apache Log Parser extrahiert strukturierte Daten aus Apache-Webserver-Logs im Combined Log Format und gibt sie als CSV-Datei aus. Zusaetzlich werden Statistiken ueber eindeutige IP-Adressen, Top-URLs und HTTP-Fehlercodes (404, 5xx) berechnet — ideal fuer die forensische Ersteinschaetzung eines Webserver-Angriffs oder fuer Incident-Response-Triage.

Parsbare Felder (Combined Log Format)

Apache Combined Log Format

Das Combined Log Format ist das Standard-Logformat von Apache und kombiniert das Common Log Format mit Referrer- und User-Agent-Informationen. Zeilenbeispiel:

192.168.1.1 - - [02/Jul/2026:12:34:56 +0200] "GET /index.html HTTP/1.1" 200 2326 "http://example.com" "Mozilla/5.0..."

Abhaengigkeiten

  • Python 3.9+ (es werden nur Standardbibliotheken verwendet: argparse, csv, re, collections, pathlib)
  • Keine externen Pakete (pip install nicht erforderlich)
  • Plattformunabhaengig (Linux, macOS, Windows)

Installation

# Ausfuehrbar machen (Linux/macOS):
chmod +x log_parser.py

# Python-Version pruefen:
python3 --version
# Benoetigt: Python 3.9 oder hoeher

Verwendung

Grundlegende Nutzung

# Apache-Log parsen, CSV auf stdout ausgeben:
./log_parser.py /var/log/apache2/access.log

# CSV in Datei speichern:
./log_parser.py access.log -o output.csv

# Mit Statistiken (auf stderr):
./log_parser.py access.log -o output.csv -s

Ausgabeformate

# Standard: CSV mit Komma-Trennzeichen
./log_parser.py access.log

# TSV (Tab-getrennt) fuer bessere Lesbarkeit:
./log_parser.py access.log --tsv

# Nur Statistiken ohne CSV (nach /dev/null umleiten):
./log_parser.py access.log -o /dev/null -s

Forensischer Mehrwert

Die vom Tool generierten CSV-Dateien lassen sich direkt in Analysewerkzeuge wie Excel, Splunk oder Python-Pandas importieren. Typische forensische Fragestellungen, die mit diesem Tool beantwortet werden koennen:

  • Brute-Force-Erkennung: IPs mit auffaellig vielen POST-Requests auf /login und Status 401/403
  • Verzeichnis-Scanning: IPs mit vielen 404-Fehlern auf ungewoehnlichen Pfaden
  • SQL-Injection: URLs mit verdaechtigen Query-Parametern (z.B. ?id=1' OR '1'='1)
  • Datenexfiltration: Auffaellig grosse Responses (Feld Size) fuer bestimmte URLs
  • Webshell-Nutzung: POST-Requests auf .php-Dateien mit untypischen Statuscodes
  • Bot-Erkennung: User-Agents die zu bekannten Scannern oder Crawlern gehoeren

Code-Struktur

Das Script log_parser.py besteht aus folgenden Komponenten:

  1. LOG_PATTERN — Regex fuer das Apache Combined Log Format
  2. parse_log_line() — Extrahiert die 7 CSV-Felder aus einer einzelnen Log-Zeile
  3. format_timestamp() — Konvertiert Apache-Zeitstempel in ISO-8601
  4. main() — Argument-Parsing, Datei-I/O, CSV-Ausgabe, Statistik-Berechnung

Einschraenkungen

  • Nur Combined Log Format (kein Common Log Format, kein JSON-Logging, kein Custom LogFormat)
  • Regex-basiert — sehr ungewoehnliche Zeilen mit Sonderzeichen koennen unerkannt bleiben
  • Keine Streaming-Verarbeitung — die gesamte Logdatei wird in den Arbeitsspeicher geladen
  • Keine GeoIP-Anreicherung (laesst sich aber einfach durch Einbindung einer GeoIP-Datenbank ergaenzen)
  • Keine Echtzeitueberwachung (Tail-Mode)
  • Timestamp-Konvertierung ignoriert Zeitzonen-Offset (wird als UTC behandelt)
