Plaso Parser Plugin Mittel

Was ist Plaso?

Plaso (Plaso Langar Að Safna Öllu, islaendisch fuer "log2timeline") ist das Standard-Framework zur Erstellung von Super-Timelines in der Digitalen Forensik. Es extrahiert Zeitstempel aus hunderten verschiedener Datenquellen (EventLogs, Registry, Prefetch, Browser-History, SQLite-Datenbanken, usw.) und vereint sie in einer einzigen, chronologisch sortierten Zeitleiste. Plaso ist das Schweizer Taschenmesser der Timeline-Forensik.

Was ist ein Plaso-Parser?

Ein Parser (auch Plugin genannt) ist eine Python-Klasse, die eine bestimmte Datei- oder Datenquelle lesen und deren zeitbezogene Informationen als EventData-Objekte extrahieren kann. Jeder Parser:

  1. Registriert sich beim Plaso-Parser-Manager
  2. Erkennt anhand einer Signatur, ob eine Datei von ihm verarbeitet werden kann
  3. Parsiert die Datei Zeile fuer Zeile (oder Binaerstruktur)
  4. Produziert EventData-Objekte mit Zeitstempel und relevanten Feldern

Die EventData-Objekte werden anschliessend von Plaso in der Storage-Datenbank gespeichert und koennen mit Tools wie psort.py oder Timesketch durchsucht und analysiert werden.

Template-Struktur im Detail

Die Datei plaso_plugin.py enthaelt eine vollstaendige Parser-Vorlage mit folgenden Bestandteilen:

KomponenteBeschreibung
EventData-KlasseDefiniert die datenbankfaehigen Felder, die der Parser produziert. Jedes Feld wird als Attribut mit DATA_TYPE registriert.
Parser-KlasseErbt von interface.FileEntryParser. Enthaelt die Hauptlogik.
NAMEEindeutiger Kurzname des Parsers (fuer CLI-Auswahl).
DATA_FORMATMenschenlesbare Beschreibung des Datenformats.
DESCRIPTIONAusfuehrliche Beschreibung fuer die Dokumentation.
GetFormatSpecification()Definiert die Signatur, anhand derer Plaso erkennt, ob dieser Parser zustaendig ist (z.B. Magic Bytes am Dateianfang).
ParseFileObject()Hauptmethode: iteriert ueber die Datei und ruft _ParseLine() fuer jede Zeile auf.
_ParseLine()Zerlegt eine einzelne Zeile/Entitaet in Felder und erzeugt ein EventData-Objekt.
SetTimestamp()Weist dem Event einen Plaso-kompatiblen Zeitstempel zu (via dfdatetime).

Installation und Verwendung

Parser im Plaso-System installieren

# 1. Parser-Datei ins Plaso-Verzeichnis kopieren:
cp plaso_plugin.py plaso/parsers/custom_app_log.py

# 2. Parser registrieren (in plaso/parsers/__init__.py oder manager eintragen):
# Der Template-Code enthaelt bereits die Registrierung:
#   manager.ParsersManager.RegisterParser(CustomAppLogParser)

# 3. Plaso mit dem neuen Parser ausfuehren:
log2timeline.py --parsers custom_app_log storage.plaso /pfad/zum/app.log

# 4. Timeline exportieren:
psort.py storage.plaso -o l2tcsv -w timeline.csv

Beispiel: Eigenes Log-Format parsen

Das Template ist fuer ein zeilenbasiertes Log-Format mit folgender Struktur ausgelegt:

2026-07-02 12:34:56,INFO,AuthModule,Benutzer 'admin' hat sich erfolgreich angemeldet
2026-07-02 12:35:01,WARN,Firewall,Verbindungsversuch von 10.0.0.99:443 abgelehnt
2026-07-02 12:35:12,ERROR,Database,Verbindung zur Datenbank fehlgeschlagen: Timeout

Jede Zeile enthaelt vier durch Komma getrennte Felder: Zeitstempel, Level, Quelle und Nachricht.

Anpassung an eigene Beduerfnisse

Um das Template fuer ein anderes Log-Format anzupassen, muessen folgende Stellen geaendert werden:

  1. EventData-Klasse: Die Attribut-Definitionen (self.event_level, self.source, self.message) an die eigenen Log-Felder anpassen.
  2. DATA_TYPE: Einen eindeutigen Bezeichner waehlen (Konvention: anwendung:modul:eventtyp).
  3. GetFormatSpecification: Die Signatur anpassen, damit Plaso die richtigen Dateien erkennt (z.B. AddNewSignature(b"MYAPP_LOG", offset=0)).
  4. _ParseLine: Die Logik zum Zerlegen einer Zeile durch eigenen Parsing-Code ersetzen (Regex, CSV, JSON, etc.).
  5. Timestamp-Parsing: dfdatetime-Klasse auswaehlen, die zum eigenen Zeitstempelformat passt (z.B. TimeElements fuer YYYY-MM-DD, PosixTime fuer Unix-Timestamps, WebKitTime fuer Browser-Zeitstempel).
Wichtig

Plaso-Parser muessen die Plaso-eigene Zeitstempel-Infrastruktur (dfdatetime) verwenden. Einfache datetime-Objekte aus der Python-Standardbibliothek werden nicht akzeptiert. Die Konvertierung erfordert das manuelle Zerlegen des Zeitstempels in seine Komponenten (Jahr, Monat, Tag, Stunde, Minute, Sekunde).

Einschraenkungen des Templates

  • Nur zeilenbasierte Log-Formate — fuer binaere Formate sind erweiterte Parsing-Methoden erforderlich
  • Keine Validierung der Parsing-Ergebnisse (fehlende Plaso-Integration im Standalone-Betrieb)
  • Einfache Signaturerkennung (nur erstes Byte des Logs)
  • Keine Unterstuetzung fuer mehrzeilige Log-Eintraege (Stacktraces, JSON)
  • Keine Behandlung von Zeichensatz-Konvertierungen (UTF-8 wird vorausgesetzt)
  • Die Datei kann nicht eigenstaendig ausgefuehrt werden — sie benoetigt eine vollstaendige Plaso-Installation
