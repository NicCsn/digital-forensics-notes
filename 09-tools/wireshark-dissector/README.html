Wireshark DFP-Dissector Mittel

Was ist ein Wireshark-Dissector?

Ein Dissector ist ein Lua-basiertes Plugin fuer Wireshark, das ein benutzerdefiniertes Netzwerkprotokoll entschluesselt und dessen Felder in der grafischen Oberflaeche darstellt. Wireshark erkennt standardmaessig hunderte Protokolle (HTTP, DNS, TLS, SMB usw.). Fuer eigene oder weniger verbreitete Protokolle kann ein Dissector geschrieben werden, der das Protokoll in seine einzelnen Felder zerlegt und lesbar macht.

DFP — Digital Forensics Protocol

Dieser Dissector implementiert das fiktive DFP (Digital Forensics Protocol) — ein einfaches binaeres Protokoll, das zu Demonstrations- und Lernzwecken erstellt wurde. Es laeuft standardmaessig auf TCP-Port 9191 und besteht aus einem festen 6-Byte-Header gefolgt von variablen Payload-Daten.

Protokollstruktur (DFP-Header)

OffsetFeldGroesseBeschreibung
0Version1 ByteProtokollversion (immer 0x01)
1Typ1 ByteNachrichtentyp (01=DATA, 02=ACK, 03=NACK, 04=HELLO, 05=RESET, 06=EVIDENCE, 07=CHECKSUM)
2Laenge2 BytesLaenge der Payload in Bytes (uint16, Big Endian)
4Checksumme2 BytesCRC-16-Pruefsumme ueber den Header
6PayloadvariabelNutzdaten (Laenge aus Feld 2)

Nachrichtentypen

  • 0x01 DATA — Allgemeine Datennachricht
  • 0x02 ACK — BEstStigung einer empfangenen Nachricht
  • 0x03 NACK — Fehlerhafte oder abgelehnte Nachricht
  • 0x04 HELLO — Initialer Handshake zwischen Client und Server
  • 0x05 RESET — Verbindung zuruecksetzen
  • 0x06 EVIDENCE — Forensische Beweisdaten (wird in Wireshark farblich markiert)
  • 0x07 CHECKSUM — Checksummen-Austausch

Felder des Dissectors

FeldnameTypBeschreibung
dfp.versionuint8Protokollversion (0x01)
dfp.typeuint8Nachrichtentyp mit benannten Werten (Wireshark zeigt z.B. "EVIDENCE" statt 0x06)
dfp.lengthuint16Laenge der Nutzdaten in Bytes
dfp.payloadbytesRohdaten der Payload
dfp.checksumuint16Pruefsumme (hexadezimale Darstellung)

Installation

  1. Plugins-Verzeichnis finden: Wireshark oeffnen → Hilfe → Ueber Wireshark → Ordner → Persoenliche Lua Plugins
  2. Datei kopieren: cp dissector.lua ~/.local/lib/wireshark/plugins/ (Linux/macOS) oder %APPDATA%\Wireshark\plugins\ (Windows)
  3. Wireshark neu starten: Nach dem Neustart erscheint DFP unter Analysieren → Aktivierte Protokolle...
  4. Testen: Mit netcat oder einem Python-Script Testdaten an Port 9191 senden und in Wireshark mitschneiden.

Verwendung nach der Installation

# Testpaket mit netcat senden (hexadezimale Rohdaten):
echo -ne '\x01\x06\x00\x0b\x00\x00Hallo Welt!' | nc localhost 9191

# Oder mit Python testen:
python3 -c "
import socket
s = socket.socket()
s.connect(('localhost', 9191))
# Version=1, Type=EVIDENCE(6), Length=13, Checksum=0
s.send(b'\x01\x06\x00\x0d\x00\x00pcap-evidence')
s.close()
"
Hinweis

Dieser Dissector ist ein Lehrbeispiel. In der Praxis wuerde ein forensisches Protokoll zusaetzliche Felder wie Timestamps, Sequenznummern und Ende-zu-Ende-Verschluesselung enthalten. Der Dissector demonstriert jedoch alle wesentlichen Bausteine: Protokolldefinition, Felddefinitionen, die Dissector-Funktion und die Registrierung bei Wiresharks TCP-Tabelle.

Dissector-Funktionsweise (Code-Uebersicht)

Der Dissector ist in dissector.lua als Lua-Script implementiert und umfasst folgende Schritte:

  1. Protocol-Objekt — Definiert das Protokoll mit Namen und Kurzbeschreibung
  2. Felder (ProtoField) — Fuenf Felder: Version, Typ, Laenge, Payload und Checksumme
  3. Dissector-Funktion — Zerlegt jedes eingehende Paket, extrahiert die Header-Felder und zeigt sie im Wireshark-Paketbaum an
  4. Port-Registrierung — Bindet den Dissector an TCP-Port 9191 (tcp_table:add(9191, dfp_proto))
  5. Heuristische Erkennung — Optional: Erkennt DFP-Pakete auch auf anderen Ports anhand des Typ-Bytes 0x01

Einschraenkungen

  • Nur TCP — keine UDP-Unterstuetzung (laesst sich aber leicht ergaenzen)
  • Keine Fragmentierung/Reassembly von grossen Payloads
  • Keine Entschluesselung (selbst bei EVIDENCE-Nachrichten werden Daten im Klartext erwartet)
  • Statische Port-Registrierung — lauscht nur auf Port 9191
  • Checksumme wird nur angezeigt, nicht validiert
