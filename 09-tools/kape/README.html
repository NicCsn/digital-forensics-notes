Kurzueberblick

KAPE (Kroll Artifact Parser and Extractor) ist ein Open-Source-Tool von Eric Zimmerman, das gezielt forensisch relevante Dateien von einem Live-System oder einem Image sammelt (Targets) und anschliessend mit Parsern verarbeitet (Modules). KAPE liest keine Dateien via RAW-Disk-Zugriff, sondern ueber das Betriebssystem — es ist also kein Imager, sondern ein gezielter Collector und Parser.

Zwei Modi: KAPE arbeitet in zwei Phasen: (1) Target-Modus = Dateien sammeln (collect), (2) Module-Modus = gesammelte Dateien parsen und analysieren. Beide Phasen koennen einzeln oder kombiniert (mit !Local) ausgefuehrt werden.

gkape — Grafische Oberflaeche

gkape.exe ist eine GUI, die den gesamten KAPE-Workflow konfigurierbar macht, ohne dass man sich CLI-Befehle merken muss. Der Workflow in gkape:

  1. gkape.exe starten (liegt im KAPE-Verzeichnis)
  2. Im Tab Targets auswaehlen, welche Artefakt-Typen gesammelt werden sollen (z.B. Registry, Event Logs, Prefetch, LNK-Dateien, Browser-Daten)
  3. Im Tab Modules auswaehlen, welche Parser auf die gesammelten Dateien angewendet werden sollen (z.B. MFT-Parser, Registry-Parser, EventLog-Parser)
  4. Source Directory angeben: Das Quelllaufwerk oder den Mount-Point des Images (z.B. C:\ oder F:\)
  5. Destination Directory angeben: Wo die Ergebnisse gespeichert werden (z.B. D:\KAPE_Output)
  6. Zusaetzlich kann ein zweites Ziel fuer die geparsten Output-Dateien angegeben werden (Module Destination)
  7. Auf Execute klicken — KAPE fuehrt zuerst alle Targets aus, dann automatisch die Module
  8. Ergebnisse nach Target-Name sortiert im Output-Verzeichnis einsehen
Empfehlung: Fuer den Einstieg reichen die Default-Targets KapeTriage und die Default-Modules !Local. Zusammen sammeln und parsen sie automatisch die gaengigsten Windows-Artefakte (Prefetch, EventLogs, Registry, Browser, USB, etc.).

CLI-Modus

Fuer Automatisierung oder Skripte wird kape.exe direkt verwendet. Die wichtigsten Parameter:

kape.exe --tsource <Quellpfad> --tdest <Zielpfad> --target <TargetName> --module <ModuleName>

# Nur Targets ausfuehren (Sammeln)
kape.exe --tsource C:\ --tdest D:\Output --target KapeTriage

# Nur Modules auf bereits gesammelte Dateien anwenden
kape.exe --msource D:\Output --mdest D:\Parsed --module !Local

# Beides kombiniert in einem Schritt
kape.exe --tsource C:\ --tdest D:\Output --target KapeTriage --mdest D:\Parsed --module !Local

# Debug-Modus mit zusaetzlichen Infos
kape.exe --tsource C:\ --tdest D:\Output --target KapeTriage --debug

# Nur auflisten, welche Targets/Modules verfuegbar sind
kape.exe --tlist
kape.exe --mlist
ParameterBedeutung
--tsourceQuellverzeichnis fuer Targets (z.B. Laufwerk C:\)
--tdestZielverzeichnis fuer gesammelte Dateien
--targetTarget-Definition auswaehlen (Name der .tkape-Datei)
--msourceQuellverzeichnis fuer Modules (meist = --tdest Verzeichnis)
--mdestZielverzeichnis fuer geparste Ergebnisse
--moduleModule-Definition auswaehlen (Name der .mkape-Datei)
--tlistListe aller verfuegbaren Targets ausgeben
--mlistListe aller verfuegbaren Modules ausgeben
--debugDebug-Ausgabe aktivieren (zeigt jeden Datei-Kopiervorgang)

Targets vs. Modules

Targets (.tkape)Modules (.mkape)
ZweckDateien kopieren / sammelnDateien parsen / analysieren
InputLive-System oder gemountetes ImageVon Targets gesammelte Dateien
Output1:1 Kopien der QuelldateienCSV, TXT, JSON — aufbereitete Ergebnisse
BeispielSAM, SYSTEM, NTUSER.DAT kopierenRegistry-Hives parsen → Benutzerliste, letzte Logins
DefinitionDateiname endet auf .tkapeDateiname endet auf .mkape
SpeicherortKAPE\Targets\ VerzeichnisKAPE\Modules\ Verzeichnis
Wichtig: Ein Module kann NICHT direkt auf ein Live-System zugreifen — es benoetigt IMMER vorher gesammelte Dateien aus einer Target-Ausfuehrung. Oder du verwendest !Local-Modules, die KAPE-Interna nutzen.

Compound Targets

Compound Targets sind vordefinierte Sets, die mehrere einzelne Targets zu einem Paket buendeln. Statt 20 Targets einzeln anzugeben, reicht ein Compound-Target-Name. Beispiele:

Compound TargetEnthaltene Einzel-TargetsEinsatzzweck
KapeTriageRegistry, EventLogs, Prefetch, LNK, JumpLists, Browser, USB, etc.Komplette Windows-Triage
KapeQuickRegistry, EventLogs, Prefetch (reduzierter Umfang)Schnelle Erstanalyse (~60 Sekunden)
_SANS_TriageSANS-empfohlene Triage-ArtefakteNach SANS-Vorgaben
AntivirusWindows Defender Logs, AV-LogsMalware-Vorfaelle

KAPE-Triage-Target — Was wird gesammelt?

Das KapeTriage-Compound-Target sammelt unter anderem:

  • Registry-Hives: SAM, SECURITY, SOFTWARE, SYSTEM, NTUSER.DAT, UsrClass.dat
  • Event Logs: Security.evtx, System.evtx, Application.evtx, PowerShell-Operational.evtx, TerminalServices*.evtx
  • Prefetch-Dateien: C:\Windows\Prefetch\*.pf — zeigt Programmausfuehrungen mit Timestamps und Run Count
  • LNK-Dateien: Recent, Quick Launch, Start Menu — Shortcut-Dateien mit Zielpfad und MAC-Zeiten
  • Jump Lists: *.automaticDestinations-ms, *.customDestinations-ms
  • Browser-Daten: Chrome, Edge, Firefox — History, Cookies, Downloads, Bookmarks
  • USB-Devices: SYSTEM\CurrentControlSet\Enum\USB, setupapi.dev.log
  • SRUM-Datenbank: SRUDB.dat — Netzwerk- und Anwendungsnutzung (Windows 8+)
  • Amcache: Amcache.hve — Programmausfuehrungen mit SHA1-Hashes
  • Shellbags: Ordnerzugriffe im Explorer
  • RecentDocs: Zuletzt geoeffnete Dokumente
  • MFT: $MFT — Master File Table (NTFS-Kern)
Vorsicht: KAPE sammelt NICHT den gesamten Festplatteninhalt, sondern nur die in den Targets definierten Dateien. Fuer ein vollstaendiges Forensik-Image wird ein Imager wie FTK Imager, dd oder Guymager benoetigt. KAPE ist ein Ergaenzungswerkzeug, kein Ersatz fuer Imaging.

Module-Beispiele

MFT-Extract

# Modul: MFTECmd (parst $MFT in CSV)
# Input: $MFT (von Target gesammelt)
# Output: MFT.csv mit allen Datei-Eintraegen, Timestamps, Pfaden

kape.exe --msource D:\Output --mdest D:\Parsed --module MFTECmd
Feld in MFT.csvForensische Bedeutung
FileNameDateiname auf dem System
ParentPathVollstaendiger Verzeichnispfad
Created0x10Erstellungszeit (MACE — Modified, Accessed, Created, Entry Modified)
LastModified0x10Letzte Modifikation
IsDirectoryDatei oder Verzeichnis?

Registry-Parse

# Modul: REGRipper (parst Registry-Hives)
# Input: SAM, SYSTEM, SOFTWARE, NTUSER.DAT (von Target gesammelt)
# Output: pro Hive eine Text-Datei mit relevanten Keys

kape.exe --msource D:\Output --mdest D:\Parsed --module REGRipper

Typische Ergebnisse aus Registry-Parsern:

  • Benutzerliste: Alle lokalen Benutzer aus SAM
  • Letzter Login: Letzter Anmeldezeitpunkt pro Benutzer
  • Autostart-Eintraege: Run/RunOnce-Keys (Persistenz)
  • Netzwerkkonfiguration: IP, DNS, Interfaces
  • USB-Historie: Angeschlossene USB-Geraete mit Seriennummer und letztem Verbindungszeitpunkt
  • Mounted Devices: Welche Laufwerksbuchstaben wurden wann zugewiesen?

EventLog-Parse

# Modul: EvtxECmd (parst .evtx-Dateien)
# Input: Security.evtx, System.evtx, etc. (von Target gesammelt)
# Output: CSV-Datei mit allen Event-Eintraegen

kape.exe --msource D:\Output --mdest D:\Parsed --module EvtxECmd

Praktische Anwendung: Windows-Triage in 5 Minuten

Szenario: Ein Windows-Client verhaelt sich verdaechtig (langsam, Pop-ups, unbekannte Prozesse). Du hast 5 Minuten vor Ort und willst eine forensische Momentaufnahme erstellen.

  1. KAPE herunterladen und entpacken: Get-ZimmermanTools PowerShell-Skript oder manuell von ericzimmerman.github.io. Auf USB-Stick kopieren.
  2. USB-Stick in den Zielrechner einstecken. KAPE laeuft portabel, keine Installation noetig.
  3. gkape.exe starten (ggf. als Administrator — empfohlen fuer Zugriff auf alle Dateien).
  4. Targets konfigurieren: KapeTriage auswaehlen (sammelt alle Standard-Artefakte).
  5. Source: C:\
  6. Target-Destination: Einen Ordner auf deinem USB-Stick auswaehlen, z.B. E:\KAPE_Output
  7. Modules: !Local auswaehlen (fuehrt automatisch alle passenden Parser aus).
  8. Module-Destination: E:\KAPE_Parsed
  9. Execute — KAPE laeuft und sammelt + parst automatisch alle konfigurierten Artefakte.
  10. USB-Stick sicher entfernen und mit der Analyse im Labor fortfahren.
Zeit sparen: Die reine Collection mit KapeTriage dauert je nach Systemlast und Datenmenge oft nur 30—90 Sekunden. Das Parsen mit !Local kann 2—5 Minuten dauern, ist aber auch offline moeglich.

CLI-Variante (skriptfaehig)

kape.exe --tsource C:\ --tdest E:\KAPE_Output --target KapeTriage --mdest E:\KAPE_Parsed --module !Local
