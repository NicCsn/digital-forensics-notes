Kurzüberblick

Die Unified Modeling Language (UML) ist die standardisierte grafische Modellierungssprache für objektorientierte Software-Systeme. Für digitale Forensiker ist UML relevant, weil sie die Struktur von Malware (Klassendiagramm), den Ablauf von Angriffen (Sequenzdiagramm), Systemarchitekturen (Komponentendiagramm) und forensische Workflows (Aktivitätsdiagramm) dokumentieren hilft — essenziell für Code-Review, Reverse Engineering und forensische Berichterstattung.

MHB-Mapping: BDF103 (Programmierung) — Grundlagen des objektorientierten Anwendungsentwurfs mit UML.

Strukturdiagramme

Klassendiagramm (Class Diagram)

Das meistgenutzte UML-Diagramm. Zeigt Klassen, Attribute, Methoden und Beziehungen. Für Malware-Analyse: Verstehen der Code-Struktur ohne Sourcecode.

+------------------+
|   C2Connector    |
+------------------+
| - server: String  |
| - interval: int   |
+------------------+
| + connect(): bool |
| + sendData(d):void|
| + beacon(): void  |
+------------------+
        ^
        |
+------------------+
| HttpConnector    |
+------------------+
| + connect(): bool |
+------------------+
BeziehungNotationBedeutung
Assoziationdurchgezogene LinieKlasse nutzt andere Klasse
Aggregationhohle Raute ◇Teil-Ganzes, Teil kann alleine existieren (Schwache Bindung)
Kompositiongefüllte Raute ◆Teil-Ganzes, Teil kann NICHT alleine existieren (Starke Bindung)
Vererbunghohles Dreieck △Subklasse erbt von Superklasse (is-a)
Interface-Impl.gestrichelt + hohles DreieckKlasse implementiert Interface (can-do)
Abhängigkeitgestrichelte Linie + PfeilKlasse benutzt andere temporär (use-a)
Forensischer Hinweis: Ein rekonstruiertes Klassendiagramm aus dekompiliertem Malware-Bytecode zeigt sofort: Welche Klasse steuert C2? Welche verschlüsselt? Wo ist die Persistenz-Logik? Das ist schneller als 5000 Zeilen Disassembler-Code zu lesen.

Komponentendiagramm

Zeigt die physische Struktur eines Systems: welche Module (.dll, .so, .jar) gibt es, wie sind sie verbunden. Für Forensik: Verstehen der Angreifer-Infrastruktur (C2-Server ↔ Proxy ↔ Malware-Binary ↔ Keylogger-DLL).

Paketdiagramm

Gruppiert Klassen in logische Pakete (Namespaces). Malware-Analyse: Welcher Code gehört zum Encryption-Modul, welcher zum Network-Modul?

Verteilungsdiagramm (Deployment)

Zeigt Hardware (Nodes) + Software (Artefakte). Forensisches Beispiel: Ransomware auf Workstation → SMB-Spread auf Fileserver → Exfiltration über VPN-Node.

Verhaltensdiagramme

Sequenzdiagramm

Zeitlicher Ablauf von Objekt-Interaktionen. Perfekt für: TCP-Handshake, Malware-Infektionskette, Ransomware-Verschlüsselungsprozess.

Opfer                          Malware                           C2-Server
  |                                |                                  |
  |────Phishing-Mail erhalten─────>|                                  |
  |                                |────beacon()────────────────────>|
  |                                |<────commands: keylog,loot────────|
  |                                |─────exfil_data()───────────────>|
  |<────Verschlüsselung startet────|                                  |

Aktivitätsdiagramm

Flussdiagramm-artig: Start → Aktivitäten → Entscheidungen → Ende. Forensischer Einsatz: Visualisierung von Incident-Response-Workflows, forensischen Prozessmodellen oder Malware-Verhalten.

Zustandsdiagramm

Zustände und Übergänge eines Objekts. Malware: Infektion → Persistenz → Idle → Active → Exfiltration → Cleanup.

Use-Case-Diagramm

Zeigt Akteure und Systemfunktionen. Forensik-Lab: "Forensiker" erstellt Image, führt Analyse durch, schreibt Bericht. Oder: "Angreifer" infiziert System, exfiltriert Daten, löscht Spuren.

Relevanz für die Digitale Forensik

Forensische AufgabePassendes UML-DiagrammNutzen
Malware-Reverse-EngineeringKlassendiagrammCode-Struktur visualisieren, Abhängigkeiten finden
AngriffsrekonstruktionSequenzdiagrammZeitlichen Ablauf des Angriffs darstellen
Incident Response PlanAktivitätsdiagrammWorkflow dokumentieren, Entscheidungslogik klar machen
Systemanalyse (Target)Komponenten-/VerteilungsdiagrammInfrastruktur verstehen: welche Server, welche Clients
Forensische BerichterstattungUse-Case-DiagrammAkteure + Aktionen für nicht-technische Stakeholder
DatenflussanalyseAktivitätsdiagrammWo fließen Daten hin? Welche Logs entstehen wo?

Werkzeuge und Praxis

Open Source

  • PlantUML: Text-zu-UML — schreibe Diagramme als Code. Versionierbar in Git, ideal für Dokumentation. @startuml ... @enduml
  • draw.io (diagrams.net): GUI-basiert, exportiert SVG/PNG/XML. Schnellster Weg für Ad-hoc-Diagramme
  • Mermaid: JavaScript-basiert, funktioniert in Markdown/GitHub. Für schnelle Sequenzdiagramme

Praxis-Beispiel: Malware-Klassendiagramm rekonstruieren

# Mit Ghidra decompilieren, dann per Hand in PlantUML übertragen:

@startuml
class MalwareCore {
  -botId: String
  -serverUrl: String
  +start(): void
  +encrypt(): void
}
class ConfigManager {
  -config: HashMap
  +loadConfig(): void
  +getValue(key): String
}
class NetworkLayer {
  +post(url, data): bool
  +get(url): String
}
MalwareCore "1" o-- "1" ConfigManager : aggregation
MalwareCore "1" --> "1" NetworkLayer : uses
@enduml
